# Supercycle Discovery Theme Taxonomy

Status: G8.1 taxonomy contract
Authority: Phase 65 G8.1
Date: 2026-05-10

## Purpose

The taxonomy gives the discovery intake layer a controlled vocabulary for theme-to-candidate mapping.

It does not create a search algorithm, ranking model, score, thesis validation, buying range, alert, provider feed, or dashboard runtime behavior.

## Theme Set

The G8.1 seed taxonomy is exactly:

```text
AI_COMPUTE_INFRA
AI_SERVER_SUPPLY_CHAIN
MEMORY_STORAGE_SUPERCYCLE
SEMICAP_EQUIPMENT
POWER_COOLING_GRID
CRITICAL_MINERALS_LITHIUM
RESHORING_FOUNDRY
DEFENSE_INDUSTRIAL
BIOTECH_PLATFORM
```

## Theme Interpretation

- `AI_COMPUTE_INFRA`: compute, accelerator, and data-center infrastructure exposure.
- `AI_SERVER_SUPPLY_CHAIN`: server, systems, networking, and integration exposure.
- `MEMORY_STORAGE_SUPERCYCLE`: memory, storage, and supply-discipline cycles.
- `SEMICAP_EQUIPMENT`: semiconductor equipment and process-control exposure.
- `POWER_COOLING_GRID`: data-center power, cooling, and grid bottlenecks.
- `CRITICAL_MINERALS_LITHIUM`: lithium and critical-mineral supply chains.
- `RESHORING_FOUNDRY`: foundry, manufacturing, reshoring, and supply-security exposure.
- `DEFENSE_INDUSTRIAL`: defense and industrial-base rebuilding exposure.
- `BIOTECH_PLATFORM`: platform-biotech exposure with repeatability evidence requirements.

## Required Evidence Shape

Each theme must define evidence requirements before a candidate can move beyond intake. Common evidence classes:

- official filing or investor-relations evidence;
- segment, backlog, demand, margin, utilization, or production evidence;
- thesis-breaker evidence;
- provider-gap disclosure for unavailable market-behavior modules.

## Boundary

Theme membership means only:

```text
candidate may deserve evidence collection
```

It does not mean:

```text
candidate is validated, ranked, scored, actionable, or recommended
```
